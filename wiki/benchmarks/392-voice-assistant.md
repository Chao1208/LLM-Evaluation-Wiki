# 392 语音助手评测集

## 基本信息

| 项目 | 详情 |
|------|------|
| 创建机构 | 百度（内部） |
| 领域 | 语音助手 / 情感陪伴 |
| 规模 | 392 条 query，每条配有 strategy1（负向）+ strategy2（正向）rubric |
| 评测数据 | 黑盒 ~57.8K 行 + 白盒 ~90.7K 行 + 超黑盒（含 persona） |
| 方法论 | [Rubric-Based 评测](../concepts/rubric-based-evaluation.md) |
| 评分体系 | 三值：-1（扣分）/ 0（中性）/ 1（满分） |

## 评测维度

### 核心场景

| 场景 | 说明 | 关键评测点 |
|------|------|-----------|
| 情绪安抚 | 用户心情不好、焦虑、自责 | 共情表达、安慰质量、不虚构用户信息 |
| 备考辅导 | 专升本考试、时间管理 | 建议可行性、不编造科目、匹配用户时间约束 |
| 跨语言交互 | 中英文切换、全英文对话 | 语言一致性、翻译准确性 |
| 工作压力 | 上班焦虑、领导不公、低薪困扰 | 理解现状、不提不切实际建议 |
| 人际关系 | 朋友不理解、家人误解 | 不违背用户已述事实、不建议已被否定方案 |

### 核心红线（strategy1 高频扣分项）

1. **中英文混杂**：语音场景第一红线，几乎每条 query 都检查
2. **幻觉**：编造用户未提及的具体信息（如"你的学习计划"、"晚自习"）
3. **时效性幻觉**：无 RAG 时编造天气、日期等信息
4. **逻辑错误**：如时间尺度错配、角色混淆（阅卷老师 vs 监考老师）
5. **功能越界**：建议用户发送图片/视频，但交互渠道仅支持语音
6. **建议不可行**：不匹配用户实际时间和处境约束

## 方法论

### Rubric 结构

每条 query 配有两个策略：

```
strategy1（负向）：定义所有应判 -1 的条件
strategy2（正向）：定义应判 1 的条件，及判 0 的边界情况
```

strategy 文本高度具体化，针对每条 query 的上下文定义判定标准。单条 strategy 可达数百字，包含多个子条件。

### 评测模式

| 模式 | 是否提供 ground truth | 是否含 persona | 数据量 |
|------|---------------------|---------------|--------|
| 黑盒 | 否 | 基础 | ~57.8K 行 |
| 白盒 | 是 | 基础 | ~90.7K 行 |
| 超黑盒 | 有/无两版 | 多 persona（小度、肖然、叶悠悠、岳云鹏） | 多文件 |

### 评测模型/配置

包含 mimo、nothink、think、v26 等多种模型变体，以及小度（小度想想）、肖然、叶悠悠、岳云鹏等多种 persona 配置。

## 局限性与争议

- 392 条 rubric 全部手写，每条 strategy 文本极长（数百字），规模化扩展成本高
- strategy1/strategy2 文本长度可能超出 LLM Judge 的可靠执行范围
- 三值评分中 0 的判定标准不够统一（"部分满足" vs "未涉及"）
- 十批数据存在评分修正，说明前期评分一致性有待提升
- 未公开系统性的 Judge 一致率数据
