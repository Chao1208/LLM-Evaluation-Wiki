# 数字人视频重建质量评测方法

## 问题定义

**任务链路**：原始视频 → 标注 caption/结构化 JSON → 生成模型依据 JSON 重建视频 → 评估原始视频与生成视频的一致性。

核心评测目标：**给定同一描述（JSON），生成视频在多大程度上"还原"了原始视频的视觉内容与动态特征？**

这与通用视频生成评测（text-to-video）有本质区别——这里有明确的 ground truth（原始视频），是一个**有参考（reference-based）** 的一致性评估问题。

---

## 一、评测维度体系

数字人视频重建不能只看一个总分，业界共识是拆分为多个正交维度分别评估：

### 1.1 身份保持（Identity Preservation）

生成人物与原始人物是否为"同一人"。

| 子维度 | 说明 |
|--------|------|
| 面部身份 | 五官特征、脸型是否一致 |
| 发型/发色 | 头发外观是否还原 |
| 肤色/肤质 | 皮肤颜色和纹理 |
| 体型/着装 | 身体比例和服饰一致性 |

### 1.2 面部表情（Facial Expression）

| 子维度 | 说明 |
|--------|------|
| AU（Action Unit）一致性 | FACS 编码的面部动作单元是否匹配 |
| 情绪还原 | 喜怒哀乐等情绪表达是否一致 |
| 表情幅度 | 微表情 vs 夸张表情是否对齐 |
| 表情时序 | 表情变化的时间节奏是否同步 |

### 1.3 唇形同步（Lip Sync）

如果涉及语音驱动：

| 子维度 | 说明 |
|--------|------|
| 唇形准确度 | 发音对应的口型是否正确 |
| 音画同步 | 唇动与音频的时间对齐 |
| 嘴部细节 | 牙齿、舌头等可见程度 |

### 1.4 头部姿态与动作（Head Pose & Motion）

| 子维度 | 说明 |
|--------|------|
| 头部朝向 | yaw/pitch/roll 角度一致性 |
| 头部运动轨迹 | 点头、摇头等动作是否还原 |
| 运动平滑度 | 是否有抖动或突变 |

### 1.5 肢体与手势（Body & Gesture）

| 子维度 | 说明 |
|--------|------|
| 手势还原 | 手部动作是否与原始一致 |
| 上半身姿态 | 肩膀、躯干的姿势 |
| 手指细节 | 手指数量、形态是否正确（常见缺陷） |

### 1.6 视觉质量（Visual Quality）

| 子维度 | 说明 |
|--------|------|
| 清晰度/分辨率 | 是否模糊或有压缩伪影 |
| 色彩一致性 | 色调、亮度是否与原始匹配 |
| 背景一致性 | 背景内容和风格是否还原 |
| 伪影 | 是否存在闪烁、撕裂、鬼影等 |

### 1.7 时序一致性（Temporal Consistency）

| 子维度 | 说明 |
|--------|------|
| 帧间稳定性 | 相邻帧是否平滑过渡 |
| 长程一致性 | 30 秒内身份/外观是否漂移 |
| 运动节奏 | 动作节奏是否与原始对齐 |

## 二、自动化客观指标

### 2.1 全参考图像/视频质量指标

这些指标以原始视频为参考，逐帧或逐片段计算。

| 指标 | 计算方式 | 优点 | 局限 | 推荐场景 |
|------|----------|------|------|----------|
| **PSNR** | 逐像素 MSE → dB | 计算快，可解释 | 与人眼感知弱相关；对微小位移过度惩罚 | 基线对比，不作主要指标 |
| **SSIM** | 亮度+对比度+结构 | 比 PSNR 更贴近感知 | 对纹理变化不敏感 | 辅助指标 |
| **MS-SSIM** | 多尺度 SSIM | 多分辨率信息 | 同 SSIM | 辅助指标 |
| **LPIPS** ↓ | AlexNet/VGG 深度特征距离 | 与人眼感知高度相关 | 需要 GPU，对齐敏感 | **核心指标，必选** |
| **DISTS** | 纹理+结构感知距离 | 对纹理合成更鲁棒 | 较新，业界采用率低于 LPIPS | 可选 |
| **VMAF** | Netflix ML 模型融合 | 工业验证充分，考虑时序 | 偏向压缩场景优化 | 视频整体质量 |
| **FVD** ↓ | I3D 特征的 Frechet 距离 | 捕捉时序分布差异 | 需大量样本才稳定（≥256） | **核心指标，评估整体分布** |

> ↓ 表示越低越好

### 2.2 数字人专用指标

| 指标 | 计算方式 | 评测维度 | 出处 |
|------|----------|----------|------|
| **CSIM** ↑ | ArcFace/FaceNet 提取人脸特征 → 余弦相似度 | 身份保持 | SadTalker, Hallo 等广泛使用 |
| **LSE-D** ↓ | SyncNet 计算唇形-音频距离 | 唇形同步 | Wav2Lip 提出 |
| **LSE-C** ↑ | SyncNet 唇形-音频置信度 | 唇形同步 | Wav2Lip 提出 |
| **AED** ↓ | 3DMM 表情系数 L2 距离 | 表情准确度 | GeneFace++ 等使用 |
| **APD** ↓ | 头部姿态（euler angles）L2 距离 | 头部姿态 | GeneFace++ 等使用 |
| **FHD** ↓ | 面部关键点 heatmap 距离 | 面部结构 | 多篇 talking-head 论文 |
| **AU-MSE** ↓ | Action Unit 强度的 MSE | 表情细粒度 | EMO、AniPortrait |
| **LMD** ↓ | 面部 landmark 距离 | 面部几何 | MakeItTalk 等 |
| **BAS** ↑ | Beat Align Score，手势与语音节拍对齐 | 手势自然度 | CaMN, SHOW 等 |

> ↑ 越高越好，↓ 越低越好

### 2.3 时序一致性指标

| 指标 | 计算方式 | 说明 |
|------|----------|------|
| **E_warp** ↓ | 光流 warp 前后帧差异 | 检测闪烁和不连贯 |
| **tOF** ↓ | 光流一致性误差 | 运动平滑度 |
| **tLP** ↓ | 感知域的时序一致性 | 结合感知特征的时序指标 |
| **FCC** ↑ | 前后向光流一致性检查 | 检测时序断裂 |

### 2.4 无参考指标（辅助）

当需要评估绝对质量（而非与参考的一致性）：

| 指标 | 说明 |
|------|------|
| **FID** ↓ | 生成帧与真实帧的分布距离 |
| **NIQE** ↓ | 基于自然图像统计的无参考质量 |
| **MUSIQ** ↑ | Google 多尺度图像质量评估 |
| **DOVER** ↑ | 视频质量评估（技术+美学双分支） |

## 三、人工主观评测方法

### 3.1 MOS 评分法（Mean Opinion Score）

**标准依据**：ITU-R BT.500 / ITU-T P.910

**操作流程**：

1. **环境控制**
   - 统一显示设备（分辨率、色彩校准）
   - 固定观看距离（建议 3-4 倍屏幕高度）
   - 安静室内环境，光线柔和无眩光

2. **评分维度设计**（建议拆分维度分别评分）

   | 维度 | 评分尺度（1-5） | 1 分 | 3 分 | 5 分 |
   |------|-----------------|------|------|------|
   | 身份一致性 | 完全不像同一人 | 有一定相似但明显差异 | 无法分辨差异 |
   | 表情还原 | 表情完全不匹配 | 大体匹配但细节有差异 | 表情完全一致 |
   | 唇形同步 | 完全不同步 | 大部分同步，偶有延迟 | 完全同步自然 |
   | 动作自然度 | 明显僵硬/异常 | 基本自然，偶有不协调 | 完全自然流畅 |
   | 视觉质量 | 严重模糊/伪影 | 可接受，轻微瑕疵 | 清晰无瑕疵 |
   | 整体一致性 | 与原始视频差异巨大 | 大体还原但有明显差异 | 高度还原 |

3. **评审员要求**
   - 人数：≥ 15 人（ITU 建议 ≥ 15，去掉异常值后 ≥ 12 有效）
   - 视力正常或矫正正常
   - 培训：先用 5-10 个 anchor 样本校准评分标准
   - 评估顺序随机化，避免顺序效应

4. **统计分析**
   - 异常评审员剔除（ITU-R BT.500 附录 2 的方法）
   - 计算 MOS 及 95% 置信区间
   - 评审员间一致性：ICC（组内相关系数）或 Krippendorff's alpha

### 3.2 成对对比法（Paired Comparison / 2AFC）

**适用场景**：对比两个模型/版本的生成结果。

**操作方式**：
- 同时播放原始视频 + 两个生成视频（A/B）
- 评审员选择"哪个更接近原始视频"
- 可加"无明显差异"选项

**分析方法**：
- Bradley-Terry 模型 → 生成胜率排序
- Elo 评分（如果有多个模型对比）
- Binomial test 检验显著性

**优势**：比 MOS 更灵活，适合快速迭代。对评审员要求较低，一致性更高。

### 3.3 ABX 法（细粒度变体）

- 展示：原始视频 + 生成视频 A + 生成视频 B
- 问题："A 和 B 哪个更接近原始？" 或 "X 是 A 还是 B？"
- 适合极其接近的方案对比

### 3.4 人工评测的实操建议（几百条 × 几十人规模）

| 项目 | 建议 |
|------|------|
| 样本分配 | 每条视频 ≥ 3 人评审，冗余保障 |
| 批次设计 | 每批 20-30 条，每批后休息 5 分钟，避免疲劳 |
| 校准批 | 每批开头插入 2-3 个 anchor 样本（不计入结果） |
| 注意力检查 | 随机插入明显好/坏的样本检测不认真评审员 |
| 平台 | 内部可用 Label Studio、自建标注平台、或简单 Google Form |
| 总时长 | 几百条 × 30s × 多维评分 ≈ 每人 2-4 小时（建议分 2-3 天完成） |

## 四、VLM-as-Judge：新兴范式

### 4.1 基本思路

用视觉语言大模型（GPT-4o、Gemini 1.5 Pro、Qwen-VL 等）作为自动评审员，输入原始视频帧和生成视频帧，让模型按评分维度打分并给出理由。

### 4.2 业界实践

| 项目 | VLM 用法 | 发表时间 |
|------|----------|----------|
| **VBench** (Huang et al.) | 用 LLM+VLM 评估视频生成的语义一致性、美学等 | 2023.11 |
| **EvalCrafter** (Liu et al.) | GPT-4V 逐维度评分视频生成质量 | 2023.11 |
| **T2V-CompBench** | 用 GPT-4o 评估 text-to-video 组合性 | 2024 |
| **GAIA** (talking avatar benchmark) | 结合自动指标 + 人工 + LLM 评分 | 2023 |
| **Hallo2** (Fudan) | 使用 GPT-4V 评估长视频数字人质量 | 2024 |

### 4.3 VLM-as-Judge 的典型 Prompt 设计

```
你是一个视频质量评估专家。你将看到两组视频帧：
- 第一组：原始视频的关键帧（参考）
- 第二组：AI 生成视频的关键帧

请从以下维度分别评分（1-5分）并给出理由：
1. 身份一致性：生成人物与原始人物是否为同一人？
2. 表情还原度：面部表情是否与原始视频匹配？
3. 姿态准确性：头部朝向和身体姿势是否一致？
4. 视觉质量：清晰度、色彩、是否有伪影？
5. 整体还原度：综合来看，生成视频与原始视频的一致性如何？

对于每个维度，请输出：
- 分数（1-5）
- 理由（1-2句）

最后给出加权总分。
```

### 4.4 VLM-as-Judge 的关键挑战

| 挑战 | 说明 | 应对策略 |
|------|------|----------|
| **视频采样** | 当前 VLM 多数不支持原生视频输入，需抽帧 | 均匀抽帧（如 1fps）+ 关键帧检测补充 |
| **时序评估弱** | VLM 对动态特征（唇同步、运动平滑）判断力弱 | 时序维度仍依赖专用指标（LSE-D、E_warp 等） |
| **评分漂移** | 不同 session 评分标准不一致 | 每批次附带 anchor 样本校准 |
| **幻觉风险** | VLM 可能"编造"不存在的问题 | 要求引用具体帧位置，设计 chain-of-thought |
| **成本** | GPT-4o 处理几百条视频的 token 成本 | 先粗筛（自动指标），VLM 评细粒度子集 |

### 4.5 VLM 与人工的一致性校准

建议流程：
1. 先让人工评测一个 50-100 条的**校准集**
2. 同一校准集用 VLM 评测
3. 计算维度级别的 Pearson/Spearman 相关系数和 Cohen's Kappa
4. 相关系数 ≥ 0.7 的维度，可以用 VLM 扩展到全量
5. 相关系数 < 0.7 的维度，仍用人工或调整 prompt

## 五、业界典型评测方案参考

### 5.1 Talking-Head 论文标配（2024-2025 主流）

大部分 talking-head 生成论文采用以下组合：

```
自动指标：CSIM + LPIPS + FID + LSE-D/LSE-C + AED + APD + FVD
人工评测：MOS（身份/唇同步/质量/自然度 4 维）, N≥20
测试集：HDTF / VFHQ / TalkingHead-1KH
```

**代表论文**：SadTalker、Hallo、EMO、AniPortrait、EchoMimic、V-Express

### 5.2 全身数字人（含手势）

在 talking-head 基础上增加：
- BAS（Beat Align Score）评估手势节拍
- FGD（Frechet Gesture Distance）评估手势分布
- 人工评测增加"手势自然度"维度

**代表论文**：SHOW、CaMN、TalkSHOW、DiffGesture

### 5.3 工业界实践特点

| 公司/产品 | 评测特点 |
|-----------|----------|
| **HeyGen** | 重视身份保持 CSIM，内部 A/B test 驱动迭代 |
| **Synthesia** | MOS + 自动指标混合，强调多语言唇形准确度 |
| **Soul Machines** | 实时交互场景，增加延迟指标 |
| **D-ID** | LPIPS + CSIM 为核心，配合用户满意度调研 |
| **腾讯/阿里/字节** | 自动指标全覆盖 + 大规模众包人工评测 |

## 七、行业趋势总结

### 趋势一：从单指标到多维度评测矩阵

早期只看 PSNR/SSIM，现在业界共识是至少覆盖 **身份+表情+唇同步+视觉质量+时序** 五大维度，每个维度有专用指标。

### 趋势二：VLM-as-Judge 快速崛起

2024 年以来，越来越多工作用 GPT-4o / Gemini 做自动评测。优势是可以评估"自然度"等难以量化的维度，劣势是时序评估能力不足。预计 2025-2026 年随着原生视频理解能力提升，VLM 评测会成为标配。

### 趋势三：人机混合评测成为最佳实践

- 自动指标做第一轮粗筛和版本间 diff
- VLM 做大规模细粒度评测
- 人工做最终校准和 corner case 评审
- 三者通过**校准集**关联起来

### 趋势四：评测与 caption 结构联动

对于 caption-driven 的重建任务，评测维度需要与 JSON 中的描述字段对齐。例如 JSON 中有 "expression: smile"，则需要专门验证生成视频中是否出现了微笑。这是传统评测框架未覆盖的，需要定制。

### 趋势五：关注长视频一致性

30 秒以上的视频评测，需要特别关注：
- 身份漂移（identity drift）—— 随时间推移人物越来越不像
- 累积误差 —— 伪影在长视频中逐渐恶化
- 分段评测 —— 将长视频切成 5s 段分别评估，再看趋势
