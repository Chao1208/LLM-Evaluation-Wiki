## 概述

LH-Bench (Metaphi.ai) 提出了面向长周期企业 Agent 任务的三层评测设计，突破二元 pass/fail 评测范式。核心发现：**专家编写的 rubric（κ=0.60）比 LLM 编写的（κ=0.46）可靠性高 30%**，关键设计原则是使用 **binary-observable boundaries**（可从 transcript 中验证的边界条件）。

**论文信息**: arXiv:2603.22744, 2026年3月, Metaphi.ai

## 核心创新

### 三层评测架构

1. **Expert-grounded rubrics**: 专家编写的 rubric，包含任务特定的技能 (skill) 维度
2. **Curated ground-truth artifacts**: 策划的真值制品（Figma 设计文件、代码数据等）
3. **Pairwise human preference**: 配对的人类偏好验证

### 两个企业环境

1. **Figma-to-code**: Agent 将 Figma 设计转为前端代码（33 tasks）
   - 4 个 process rubric: Design Inspection (0.30), Token Extraction (0.25), Component Architecture (0.25), Build Verification (0.20)
2. **Programmatic content**: Agent 从文档集合生成教学内容（183 chapters, 41 courses）
   - 5 个 skill rubric: content selection, narrative structure, visual hierarchy, information density, source grounding

### 并行三 Judge 评估

| Judge | 输入 | 评估内容 |
|-------|------|----------|
| Trajectory | Transcript + tool traces | Planning, recovery |
| Process | Transcript + skill rubric | Workflow compliance |
| Output | Ground truth + screenshots | Visual fidelity |

## 关键实验数据

### 专家 vs LLM rubric (Table 8)

| 指标 | v1.1 (LLM, 8 rubrics) | v1.2 (Expert, 4 rubrics) |
|------|------------------------|--------------------------|
| Mean pairwise κ | 0.46 | **0.60** |
| Mean variance | 0.25 | **0.10** |

**+0.15 κ 提升来自**: (1) binary-observable boundaries, (2) 更少但更精准的维度, (3) transcript evidence 锚点

### Rubric 设计原则

- **Binary-observable**: "Did the agent create a token file before writing components?" 比 "Did the agent demonstrate good planning?" 更可靠
- **Fewer is better**: 4 个精准维度 > 8 个模糊维度
- **5分量表 > 3分量表**: 减少 49% 平局，提升区分力

### SKILL.md ablation (Table 7)

Expert workflow guidance (SKILL.md) 平均提升 +0.33 分（pooled across harnesses），验证了专家知识注入的价值。

### 恢复能力分析

- 70% 的错误被 Agent 自动恢复
- Tool call failures 占 71% 的错误，恢复率 66.5%
- Build errors: 100% 恢复率
- Config errors: 仅 16.7% 恢复率

## 对我们研究的核心启示

1. **Binary-observable boundaries**: 这是 rubric 设计的黄金标准——可从证据中直接验证的标准比主观判断可靠得多
2. **Expert rubric >> LLM rubric**: 虽然我们目标是自动化，但需要以专家 rubric 质量为对标
3. **5分量表**: 对我们的评分设计有直接参考——三值评分可能不够，5 级更有区分力
4. **Fewer, sharper rubrics**: 与 AdaRubric 的 N=5 结论一致
5. **Multi-tier evaluation**: 过程评估 + 结果评估的结合值得借鉴
6. **SKILL.md 知识注入**: 类似我们通过 prompt/specification 注入专家知识的思路
