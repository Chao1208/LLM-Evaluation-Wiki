## 概述

RubricRAG 提出通过检索增强生成 (RAG) 方式自动生成评测 rubric。核心思想是从已有的领域知识库中检索相似的评测 rubric，用于指导新 rubric 的生成，而非从头让 LLM 创造评测标准。

**论文信息**: arXiv:2603.20882, 2026年3月

## 关键发现

1. **检索增强 rubric 生成**: 利用已有 rubric 库作为参考，通过相似性检索提供 few-shot examples，指导 LLM 为新任务生成高质量 rubric
2. **领域知识复用**: 同一领域内的评测标准具有高度相似性，RAG 范式可有效利用这种结构化相似性
3. **与 AdaRubric 互补**: RubricRAG 关注 rubric 的知识基础（从哪里来），AdaRubric 关注 rubric 的结构设计（如何组织）

## 方法论

- **检索阶段**: 给定新的评测任务描述，从 rubric 知识库中检索语义最相似的已有 rubric
- **生成阶段**: 将检索到的 rubric 作为上下文，引导 LLM 生成针对当前任务的新 rubric
- **优势**: 减少 LLM 的"创造"负担，提高 rubric 的领域适配性和一致性

## 对我们研究的启示

- **rubric 库构建**: 可以为医疗领域建立 rubric 库，复用跨 case 的评测标准
- **与 specification-driven 方法结合**: RAG 提供的相似 rubric 可作为 "anchor examples"，增强评测规范的具体性
- **冷启动问题**: 当领域 rubric 库为空时需要其他方法（如 AdaRubric 的自适应生成）
