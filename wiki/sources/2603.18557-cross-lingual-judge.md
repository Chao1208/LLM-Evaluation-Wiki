## 概述

本文（CISPA/Amazon）提出 Universal Criteria Set (UCS) 框架，通过评测分解实现跨语言 LLM Judge 迁移。核心思路：将评判分解为语言无关的 criteria 向量，训练轻量迁移模块在英文数据上，零样本迁移到其他语言。在 11/12 评测语言上达到最优或并列最优。

**论文信息**: arXiv:2603.18557, 2026年3月, CISPA Helmholtz Center / Amazon

## 核心方法

### 三阶段框架

1. **Criteria Set Generation**: 从任务描述生成语言无关的评测 criteria 集合 (UCS)
   - 输入: 任务描述 t → 生成评测概念 C = {c1, ..., cm}
   - 每个概念生成子问题 Q_j，最终 UCS = 所有问题的并集
   - **Criteria 用英文生成**（LLM 英文推理更稳定），但适用于所有语言

2. **Criteria Set Evaluation**: 对每个样本，LLM 逐条 criterion 打分（1-10）
   - 产生 criteria-response 向量 z(x) = [z1(x), ..., zk(x)]
   - 这是语言无关的中间表示

3. **Cross-Lingual Transfer**: 在英文标注数据上训练轻量预测器
   - 输入: concept-level 聚合后的向量 s(x) = [s1(x), ..., sm(x)]
   - 浅层神经网络（32 hidden units），仅需英文数据
   - 零样本迁移到其他语言

### Concept-Level Aggregation

```
s_j(x) = (1/|Q_j|) * sum_{q_i in Q_j} z_i(x)
```

在 concept 级别聚合（而非 criterion 级别），降低维度、提升迁移鲁棒性。

## 关键实验数据

### 跨语言评测 (Balanced Accuracy, Qwen-235B)

| 方法 | MEMERAG (Avg 5 lang) | mFACE (Avg 8 lang) |
|------|---------------------|---------------------|
| Zero-shot | ~78% | ~72% |
| CoT | ~77% | ~73% |
| ChatEval | ~71% | ~63% |
| CheckEval | ~80% | ~76% |
| RocketEval | ~79% | ~73% |
| **UCS (EN)** | **~83%** | **~78%** |

### 迁移模块比较 (Table 2)

| 方法 | MEMERAG | mFACE |
|------|---------|-------|
| LogReg | 80.9 | 78.2 |
| SVM | 78.5 | **79.0** |
| NN | **81.5** | 78.9 |

浅层神经网络最优，但 LogReg/SVM 也很有竞争力——说明 criteria 表示的信号是**线性可分**的。

### 样本效率

20-30% 英文训练数据即可达到接近最优性能，对低资源场景友好。

## 对我们研究的启示

1. **中文医疗评测的跨语言问题**: 可用英文 rubric 标准评估中文医疗回答——criteria 在语言间可迁移
2. **Criteria 向量作为中间表示**: 支持我们的 "specification as program" 思路——rubric 产出的不是一个分数，而是一个结构化的判断向量
3. **Concept-level aggregation**: 粒度选择很重要——太细（逐 criterion）噪声大，太粗（整体评分）信息损失
4. **轻量迁移**: 无需大模型微调，只需简单分类器，符合我们 "无训练" 的约束
5. **评测分解 + 迁移 = 通用性**: 与 DeCE 的分解思路互补，但增加了跨语言维度
