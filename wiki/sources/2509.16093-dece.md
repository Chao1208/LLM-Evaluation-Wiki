## 概述

DeCE (Decomposed Criteria-based Evaluation) 提出将 LLM 评测分解为 **precision（精确度/准确性）** 和 **recall（覆盖度/完整性）** 两个正交维度，各自进一步拆分为细粒度 criteria。在多个任务上，DeCE 的人类相关性从传统整体评分的 r=0.12 提升至 r=0.78。

**论文信息**: arXiv:2509.16093, 2025年9月

## 关键发现

1. **Precision-Recall 分解**: 传统整体评分混淆了"回答是否准确"和"回答是否全面"，分开评估后相关性大幅提升
2. **r=0.78 vs r=0.12**: 分解后的评测与人类判断的 Pearson 相关性提升 5.5 倍
3. **Precision 细分**: 事实准确性、相关性、逻辑一致性等
4. **Recall 细分**: 关键信息覆盖、要点完整性、细节充分性等
5. **通用框架**: 不限于特定任务类型，可应用于问答、摘要、对话等

## 方法论

```
传统评测:
  Response → LLM Judge → Score (1-5)  [r=0.12]

DeCE 评测:
  Response → Precision 评估 → {accuracy, relevance, consistency, ...}
           → Recall 评估   → {coverage, completeness, detail, ...}
           → 加权聚合        → Final Score  [r=0.78]
```

### 评测流程

1. **Criteria 生成**: 从任务描述中导出 precision 和 recall 的具体 criteria
2. **逐条评估**: LLM judge 对每条 criterion 独立打分
3. **结构化聚合**: 按权重聚合为最终分数

## 重要数据

| 方法 | Pearson r | 说明 |
|------|-----------|------|
| 传统整体评分 | 0.12 | 单一 prompt 让 judge 直接打分 |
| DeCE (precision only) | ~0.55 | 仅评估准确性 |
| DeCE (recall only) | ~0.50 | 仅评估覆盖度 |
| DeCE (full) | 0.78 | precision + recall 组合 |

## 对我们研究的启示

- **核心方法论启发**: Precision/Recall 分解可直接应用于医疗评测——"回答是否正确"和"回答是否全面"是医疗咨询的两个核心维度
- **与 HealthBench 正负分体系呼应**: HealthBench 的 positive criteria（应该包含的内容）≈ recall，negative criteria（不应包含的内容）≈ precision 的反面
- **与 AdaRubric 互补**: DeCE 提供分解框架，AdaRubric 提供自适应维度生成，两者可结合
- **评测规范的结构化**: 支持我们 "specification as program" 的方向——将评测分解为可执行的子判断
