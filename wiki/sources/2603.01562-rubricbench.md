## 概述

RubricBench 评估 LLM 生成的评测 rubric 与人类专家设计的 rubric 之间的对齐程度。提供了系统化的方法来衡量自动生成 rubric 的质量，包括覆盖度、精确度和可操作性等维度。

**论文信息**: arXiv:2603.01562, 2026年3月

## 关键发现

1. **LLM 生成的 rubric 与人类 rubric 存在系统性偏差**: 模型倾向于生成过于泛化的标准，缺乏领域特异性
2. **rubric 质量直接影响评测质量**: 质量差的 rubric 导致 judge 一致性下降
3. **对齐度量方法**: 提供了多维度的 rubric 质量评估框架
4. **可作为 rubric 生成方法的元评测工具**: 衡量不同 rubric 生成方法（手工/LLM/RAG/自适应）的质量

## 评测维度

- **覆盖度 (Coverage)**: rubric 是否涵盖了任务的所有关键评测方面
- **精确度 (Precision)**: rubric 的描述是否足够具体、无歧义
- **可操作性 (Actionability)**: 评测者是否能根据 rubric 做出一致的判断
- **正交性 (Orthogonality)**: 各维度之间是否独立、无重叠

## 与其他工作的关系

| 工作 | 关注点 |
|------|--------|
| RubricBench | rubric 本身的质量评估 |
| RubricEval | judge 使用 rubric 时的判断准确性 |
| AdaRubric | rubric 的自动生成方法 |
| RubricRAG | rubric 的检索增强生成方法 |

## 对我们研究的启示

- **rubric 质量评估框架**: 可用于评估我们的 specification-driven rubric 的质量
- **人类对齐是终极标准**: 再好的自动生成方法，最终要与人类专家标准对齐
- **AdaRubric 人类评估的补充**: AdaRubric Table 10 显示自动生成 rubric 在 relevance/orthogonality/completeness 上接近专家水平（4.3/4.2/4.1 vs 4.6/4.4/4.5）
